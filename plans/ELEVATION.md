# Release closure — Invisible Interfaces

The former elevation plan described a six-scene concept that no longer exists. This file now records the canonical closure criteria for the five-scene release.

## Completed direction

- Entrusted absence is the signature interaction.
- Return includes a tangible original/restored comparison.
- Work, authority, reversal, and attention receipts arrive together.
- Attention data is session-scoped, disclosed, returned, and deleted with the tab.
- A labeled accessibility fallback exists for unreliable visibility state.
- Historical captions describe relationships rather than false chronology.
- Header, rail, motion, type, contrast, mobile layout, metadata, and code splitting follow the current visual system.
- About is available from the early navigation and after completion.
- All documentation and tests must refer only to scenes 0–4.

## Final release checklist

- Production build passes.
- First-load route JavaScript is at or below 160 kB.
- Mojibake scan returns zero.
- Desktop and mobile screenshots represent the current release.
- Keyboard, reduced-motion, early return, full return, fallback, discard, About, and 404 paths pass.
- No obsolete six-scene screenshots or test expectations remain.
