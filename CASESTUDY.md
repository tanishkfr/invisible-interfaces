# Invisible Interfaces - Case Study

**A measured essay about attention leaving the interface.**

Invisible Interfaces is not a history lesson about UI. It is an interactive argument that spends less and less of the visitor's attention until the interface nearly disappears, then returns a receipt of what it took.

## Research Question

How did interfaces disappear from conscious attention, and what did that disappearance cost or return to the user?

## Core Thesis

Interfaces did not disappear. They moved out of attention. That movement was designed, decision by decision.

## Contribution

Invisible Interfaces contributes a phenomenological interaction structure:

- the visitor first experiences explicit computational friction
- the same task is softened across interaction paradigms
- the system begins acting before the visitor asks
- the essay measures the visitor's attention locally
- the ending reads that trace back without sending it anywhere

The argument is inherited from ubiquitous computing and no-interface writing. The staging is the contribution.

## Interaction Grammar

**Endure friction -> watch friction collapse -> let software act -> receive the attention receipt**

The project should not become a general ethics-of-AI piece. Its unique territory is attention: cost, disappearance, and return.

## Cognitive Verb

**Notice**

The project succeeds when the reviewer begins noticing invisible interfaces in ordinary life after leaving.

## Emotional Arc

Irritation: "The machine demands too much."  
Relief: "The same task asks less of me."  
Unease: "The system is acting before I ask."  
Recognition: "The interface disappeared from attention, not from existence."

## Reviewer Takeaway

Invisible Interfaces proves Tanishk can stage an argument as a lived temporal experience.

Two weeks later, the reviewer should remember:

> Invisible Interfaces made me feel the history of UI as a transfer of attention.

## What Only This Project Teaches

Invisible Interfaces is the only project in the portfolio that makes attention the subject, material, and evidence of the work.

Pentimento also involves software reading a person, but Pentimento is about authorship and correction. Invisible Interfaces is about attention cost.

## Closest Portfolio Neighbor

**Pentimento**

The overlap is mostly authorial: both care about what software does with a person.

The difference:

- Invisible Interfaces asks, "Where did my attention go?"
- Pentimento asks, "Who gets authority over a machine's interpretation of me?"

If Invisible Interfaces is framed as contestability, it weakens Pentimento. If it is framed as attention choreography, it stands alone.

## 30-Second Reviewer Framing

Invisible Interfaces should introduce itself with this claim:

> This essay makes you operate an attention-hungry machine, then gradually removes the interface until the system can tell you what your attention cost.
