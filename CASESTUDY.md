# Invisible Interfaces

> **Context:** Self-directed research-through-design exhibition
> **Year:** 2026
> **Duration:** 2026
> **Role:** Independent interaction designer, writer, and engineer
> **Status:** Complete v1.0 browser-based exhibition
> **Tools:** Next.js, TypeScript, Motion, Tailwind CSS, Page Visibility API
> **AI disclosure:** AI-assisted ideation, critique, and code iteration; final concept selection, design decisions, writing, editing, and authorship by Tanishk Salagame.
> **Live:** https://invisible-interfaces.vercel.app/
> **Source:** https://github.com/tanishkfr/invisible-interfaces

## Attention is demanded

A terminal asks the visitor to find a beach photograph. It requires syntax, commands, error recovery, and continuous attention.

The project begins with a question: what changed when software stopped requiring us to watch it work?

## Friction collapses

The same photograph moves through pointing, touching, search, and prediction. These are relationships to a task, not a claim about invention order. The task remains stable while visible labor falls.

The attention ledger makes the reduction legible while refusing to call it free: fewer visible decisions can relocate judgment into systems the person cannot see.

## The system remembers

Software begins acting before an explicit request. A sign-in fills itself, a reply suggests itself, a calendar listens, and the photograph returns as a memory. Relief and unease arrive together.

## The task is entrusted

The photograph returns as a consequential request: locate the source, repair the damaged scan, preserve the original, and stage a private copy.

The interface refuses to run while the visitor watches. Each interval out of view advances one bounded movement of work. Returning early pauses what remains. A disclosed demonstration path prevents the interaction from becoming an accessibility or browser-capability trap.

## Accountability appears

Completion reveals the restored photograph beside the damaged source. A keyboard- and touch-accessible comparison control lets the visitor inspect the intervention.

The return receipt states:

- what was located, repaired, compared, and staged;
- what remained untouched;
- what was not transmitted;
- what the system was not allowed to infer or do;
- how the restored copy can be discarded.

A second receipt returns the essay’s own observations—opening patience, terminal effort, reverse scrubs, and navigation use. Those measurements remain in session storage and disappear when the tab closes.

## Contribution

Invisible Interfaces demonstrates absence as an interaction event rather than empty waiting. It pairs invisible work with accountable return instead of continuous supervision or a theatrical agent dashboard.

The image to remember is a gallery piece that cannot finish while anyone is looking at it—and that has to show its work when the person returns.

## Limits

The browser stages authored work; it does not repair a real archive or run a deployed agent. The artifact investigates an experiential relationship among attention, delegation, authority, and return. It does not claim participant evidence or operational safety validation.
