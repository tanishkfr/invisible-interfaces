# Invisible Interfaces

> **Context:** Self-directed research-through-design exhibition
> **Year:** 2026
> **Duration:** Ongoing in 2026
> **Role:** Independent interaction designer, researcher, writer, and engineer
> **Status:** Working browser-based exhibition
> **Tools:** Next.js, TypeScript, Motion, Tailwind CSS, Page Visibility API
> **AI disclosure:** AI-assisted ideation, critique, and code iteration; final concept selection, design decisions, writing, editing, and authorship by Tanishk Salagame.
> **Live:** https://invisible-interfaces.vercel.app/
> **Source:** https://github.com/tanishkfr/invisible-interfaces

## 00:00 — Attention is demanded

A terminal asks the visitor to find a beach photograph. It requires syntax, commands, error recovery, and continuous attention.

The project’s question begins here: **what changed when software stopped requiring us to watch it work?**

## 00:90 — Friction collapses

The same photograph moves through pointing, touching, search, and prediction. The task remains stable while the person’s visible labor falls.

This is not presented as a triumphal history. Each reduction relocates coordination and makes different decisions harder to see.

## 02:10 — The system remembers

Software begins acting before the explicit request. Relief and unease arrive together: less effort does not mean less design.

## 03:00 — The task is entrusted

The photograph returns as a more consequential request: locate the source, repair the damaged scan, preserve the original, and stage a private copy.

The interface refuses to run while the visitor watches.

The visitor entrusts the task and leaves the tab. Absence is causal. Each interval out of view advances one bounded movement of work. Returning early pauses the remaining work.

## Return — Accountability appears

Once the task is complete, the visitor returns to the result and its receipt at the same moment:

- what was located, repaired, compared, and staged;
- what remained untouched;
- what was not transmitted;
- what the system refused to infer;
- how the restored copy can be discarded.

Only then does the exhibition’s ending exist. Scrolling cannot bypass the thesis.

## Emotional passage

**Irritation → relief → unease → surrender → recognition.**

The decisive interaction happens offscreen, but the design responsibility does not. The final interface appears because the visitor left it.

## Contribution

Invisible Interfaces demonstrates absence as an interaction event rather than empty waiting. It pairs invisible work with accountable return instead of continuous supervision or a theatrical agent dashboard.

## Limits

The browser stages authored work; it does not repair a real archive or run a deployed agent. Visibility state and the ledger remain local to the tab. No analytics or remote attention data is collected. The artifact investigates an experiential relationship among attention, delegation, and return; it does not claim participant evidence or operational safety validation.

The image to remember is a gallery piece that cannot finish while anyone is looking at it.
