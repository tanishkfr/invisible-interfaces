# Invisible Interfaces — product specification

Version 1.0

## Thesis

Interfaces have not disappeared; they have moved out of attention. Reduced visible operation does not remove design responsibility. Invisible work owes the person an accountable return.

## Canonical information architecture

0. The Forgotten Interfaces
1. Demanding Attention
2. The Great Simplification
3. When Software Started Remembering
4. The Entrusted Task and Return

The About route is a reading room, not a sixth scene.

## Scene requirements

### 0 — The Forgotten Interfaces

- Create curiosity through two statements and familiar peripheral artifacts.
- Preserve the opening sequence when hurried or reduced.
- Give screen-reader users an equivalent list of the examples.
- Let the header arrive with the first statement, never before it.

### 1 — Demanding Attention

- Require a real keyboard or touch-keyboard interaction.
- Teach through literal responses, then provide mercy before frustration becomes abandonment.
- Persist terminal outcome only for the current tab session.
- Announce complete machine lines to assistive technology, not every animated character.

### 2 — The Great Simplification

- Keep one photograph continuous across five interaction relationships.
- Let the visitor scrub backward to compare.
- Describe attentional distance rather than a false invention chronology.
- Show that available attention is not the same as free attention.
- Provide a discrete reduced-motion rendering.

### 3 — When Software Started Remembering

- Present four pre-emptive behaviors without requiring clicks.
- Return the visitor’s own terminal outcome when available.
- Keep one active beat legible at a time.
- Preserve equivalent semantics with reduced motion and screen readers.

### 4 — The Entrusted Task and Return

- Advance work only during real hidden-tab intervals.
- Pause remaining work on early return.
- Provide a clearly labeled accessible demonstration when visibility cannot be observed.
- Reveal an inspectable original/restored result, work receipt, authority boundary, and reversible discard.
- Return the local attention ledger and state its retention accurately.
- Do not expose the ending or About-after-ending doorway before completion.

## Interaction system

The signature interaction is absence as input. Every earlier interaction establishes the attention relationship that the final interaction reverses.

Decorative motion, shadows, hidden hover meaning, pointer-precision requirements, and undisclosed measurement are prohibited.

## Accessibility

- Native landmarks and headings.
- Visible two-pixel focus treatment.
- Skip link.
- Complete keyboard and touch access.
- Screen-reader equivalents for visual evidence.
- Reduced-motion variants for every timed or scroll-linked sequence.
- No meaningful text below WCAG AA contrast after opacity is applied.
- No completion path that depends exclusively on browser visibility support.

## Performance and release targets

- Static export.
- No runtime network requests or analytics.
- First-load JavaScript at or below 160 kB for the interactive route.
- No horizontal overflow at 320, 375, 768, or 1280 CSS pixels.
- Production build, copy-integrity scan, and end-to-end walkthroughs must pass.
- Metadata, canonical URL, Open Graph image, About route, and 404 route must export correctly.
